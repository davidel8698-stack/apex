# R10-002: Intentional-no-test taxonomy.
#
# Format:
#   - One basename per line (with extension).
#   - Comment lines start with `#` and document the rationale for the
#     entry on the immediately FOLLOWING non-comment line. The
#     coverage_scan exemption check ignores comment lines and matches
#     basenames literally.
#
# Categories (per REMEDIATION-PLAN-R10.md R10-002 question 10):
#   (a) user-paste-driven prose with no machine contract — slash
#       commands whose body is prompt-engineering text consumed by
#       Claude Code at runtime; correctness is measured by user
#       outcome, not by an executable assertion.
#   (b) documentation-only schema with no machine contract — JSON
#       schemas referenced only by docs, not loaded or validated by
#       any hook or script at runtime.
#   (c) deferred for a later R-item — currently empty.
#
# Adding an entry:
#   1. Pick a category (a/b/c) and document it in the rationale comment.
#   2. Place a `#`-prefixed rationale line, then the basename on the
#      next non-comment line.
#   3. The runner banner ⚠️ count drops by one on the next self-test run.
#
# Removing an entry (when a real test is authored):
#   1. Add the test under framework/tests/test-<name>.sh.
#   2. Delete the entry's rationale comment + basename here.
#   3. The runner banner ⚠️ count is unchanged (the new test now
#      provides coverage; the entry no longer needs an exemption).
#
# Spec anchors:
#   "Verification universal, not TDD universal. Test architecture is
#     its own discipline with veto power."
#   "Honest scope over marketing scope."
#   "Proof-of-process beats proof-of-promise."
